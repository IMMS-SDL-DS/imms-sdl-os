# 📄 ChemOS 2.0 논문 요약

> Sim et al., *Matter*, 7(9), 2959–2977, 2024  
> DOI: 10.1016/j.matt.2024.04.022

---

## 핵심 한 줄

**"실험실을 OS로 본다 — 장비·데이터·실험 계획을 하나의 운영체제처럼 통합 관리하는 오케스트레이션 플랫폼"**

---

## 주요 개념

### 1. 실험실 = 운영체제 (핵심 철학)
- 컴퓨터 OS가 CPU·메모리·앱을 관리하듯
- ChemOS 2.0은 **장비·DB·실험 계획**을 통합 관리
- "fog computing device"가 OS 커널 역할

### 2. 모듈형 DB 구조
```
device      → 연결된 장비 목록
job         → 실행 중인/예정된 실험 작업
devicelog   → 장비별 실행 기록
(+ 장비별 특화 테이블)
```
- **어떤 SDL에도 이식 가능**한 범용 구조
- 실험 DB + 시뮬레이션 DB 분리 (AiiDA 연동)

### 3. SiLA2 기반 장치 통신
- 표준 프로토콜로 장비와 소프트웨어 연결
- 소켓 서버로 연결 손실 시 장비 안전 보장
- 장비 없을 때 **시뮬레이터**로 테스트 가능

### 4. 점진적 도입 가능
- 처음부터 전체 자동화 불필요
- 장비 하나씩 추가하며 확장

---

## 실제 검증
- 유기 고체 레이저 소재 발견 워크플로우에 적용
- DFT 계산 + 실험 오케스트레이션 통합
- 폐루프 closed-loop 운영 실증

## 우리 연구에 적용할 부분

| ChemOS 2.0 | 우리 프로젝트 |
|------------|--------------|
| device/job/devicelog 테이블 | MOF DB 스키마 설계 기준 |
| SiLA2 장치 통신 | XRD 장치 연결 참고 |
| 시뮬레이터 테스트 | 장치 없을 때 개발 환경 |
| 점진적 도입 | MOF 장비 하나씩 연결 |
