## 핵심 한 줄

**"실험을 모듈 단위 task로 쪼개고, 자원 충돌 없이 동시에 여러 워크플로우를 돌리는 Python 기반 SDL 운영체제"**

---

## 주요 개념

### 1. 재설정 가능한 워크플로우 모델
- 실험을 **모듈형 task**로 분리 → 레고처럼 조립
- 실험 순서가 바뀌어도 전체를 다시 짤 필요 없음
- 그래프(DAG) 기반으로 task 간 의존성 관리

### 2. 자원 예약 메커니즘 (Resource Reservation)
- 장비가 여러 실험에 동시 요청될 때 **충돌 방지**
- "지금 이 장비는 A 실험이 점유 중" → 자동 대기열 관리
- 데드락(deadlock) 방지 알고리즘 내장

### 3. DB 백엔드: MongoDB
- NoSQL → 유연한 스키마 (실험마다 다른 필드 허용)
- 샘플 위치, 메타데이터 실시간 추적
- 상태 관리: 대기 / 실행 중 / 완료 / 실패

---

## 실제 검증
- **A-Lab (LBNL)**에서 1.5년간 **3,500+ 샘플** 합성에 적용
- 하루 최대 149개 샘플 처리 (2024.02.09)
- 고체 상태 합성에 특화
